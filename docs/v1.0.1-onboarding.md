# v1.0.1 — Onboarding seeds & provider UX

发布日期：2026-05-31

第一版（v1.0）能跑通所有功能，但是空数据库 + 全靠用户从 0 配，门槛偏高。
v1.0.1 是一次纯 UX 补丁：装完打开就有样例可点，配模型只用点一次。

## 新功能

### 1. 一键种子（首次启动自动）

`src/features/seedTemplates.ts`，在 `bootstrap()` 末尾调用。每一段只在
对应表为空时种入，**已有数据永不覆盖**，删了样例也不会被重新塞回。

**Skills (11)** — 全部带 `source_url` 写在 `metadata_json` 里：

| 来源 | 技能 |
|---|---|
| awesome-chatgpt-prompts | Linux 终端 / JavaScript 控制台 / 代码审查官 / 中英翻译 / 技术文档作者 / Excel 公式助手 / 财务分析师 / 面试官 / 查重风格甄别 |
| anthropics/skills 改写 | 写作风格守护 |
| SillyTavern 社区 | 角色一致性守护 |

**Cards (3)**：小绿（闲聊）、Translator Pro（工作）、阿狸（LoL 公开 lore 改写，
`extensions.shanhaijing_inspiration` 标了出处）。

**Agents (7)** — 故意把 `default_provider_id` / `default_model` 留空，
让用户首次去 Agents 面板的"批量配置"块一次性套全：

- 纯职业：程序员小张 / 文案小李 / 私人秘书 / 财务老王 / 翻译官
- IP × 职业：总裁秘书·阿狸（card=阿狸 + 秘书 skill + persona_text 叠加层）、
  程序媛·阿狸（card=阿狸 + 开发 skill）

**Conversations (5)** — 装完直接能点：

|---|---|---|
| private | 和小绿瞎聊 | 小绿 |
| private | 和阿狸聊聊今天的安排 | 总裁秘书·阿狸 |
| casual  | 下班放松群 (≤30 轮) | 小绿 + 程序媛·阿狸 |
| work    | 接需求·出技术文档 | 程序员小张 + 文案小李 |
| work    | 总裁办公室·周一例会 | 秘书阿狸 + 私人秘书 + 财务老王 |

### 2. Providers 面板：Modal 化

之前是"点新建后表单出现在列表下方，用户根本没往下滚 → 以为没字段"。
现在：

- "+ 新建 provider" 和"编辑"都改成弹 Modal，**名称 / Base URL / API Key /
  协议**一目了然。
- 列表为空时显示明确空态提示，不再让用户对着空白发呆。
- `kind` 下拉补上 `anthropic` 选项（之前 `Provider["kind"]` 类型已有 anthropic
  但下拉漏了）。
- 列表项里没填 key 的 provider 会有"（未填 API Key）"小标记。
- 进面板自动 `reload()` 一次，避免列表偶发为空。

### 3. AgentsPanel：批量配置块

种子塞了 7 个 agent，全部 `provider/model` 为空。一个个点开太蠢。新加的"批量
配置"块在列表上方：

- 状态行：`有 N / M 个 agent 还没选 provider/model`。
- 一行下拉：选 provider → 自动加载该 provider 缓存的模型列表；可手动输入兜底。
- 复选框 `覆盖已配置的 agent`：默认只动空的，勾上会扫全部，方便整组换模型。
- 一键 `套用到 N 个 agent` → 写 `default_provider_id` + `default_model`。

### 4. 文案清理

- `docs/v1.0-release-notes.md` 删掉 "treats LLM agents like friends, not tools"
  这种营销话术。
- `src/llm/prompt.ts` system prompt 里的 `"a human friend named ..."` 改为
  `"a human user named ..."`。

## Build

```bash
pnpm install
pnpm exec vite build        # web bundle
pnpm tauri build            # desktop installers (Win / macOS / Linux)
```

Windows 端两个安装包还是 `.msi`（WiX，5.95 MB）和 `-setup.exe`（NSIS，4.39 MB），
都自动注册到"应用和功能"里，可正常卸载。

## 用户升级流程

老用户从 v1.0 升上来，已有的 providers / agents / skills / cards / conversations
**完全保留**，新种子**不会**塞进非空表，只有"全空"的那几张表才会被种入样例。

如果想要这次的样例数据，可以：

1. 关闭 app。
2. 在系统数据目录里把 `shanhaijing.db` 重命名（备份）。
3. 重启 app。bootstrap 看到表全空，会种入完整样例。
