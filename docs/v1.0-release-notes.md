# v1.0 — First public release

山海经 (Shanhaijing) 1.0 ships every feature on the original roadmap from v0.1
through v0.7. This is the line we promised: a local-first desktop chat client
that treats LLM agents like friends, not tools.

## Feature recap (what 1.0 has)

### Core (v0.1–0.2)
- Tauri 2 + React 19 + TypeScript + Tailwind v4
- SQLite via `tauri-plugin-sql` with versioned migrations
- Three-pane shell: AgentRail / ConversationList / ChatPane
- Provider CRUD with AES-GCM encrypted API key storage
- Remote model list (OpenAI-compatible `/v1/models`)
- User Persona CRUD
- Agent CRUD (provider + model + sampling params)
- Private 1v1 conversation: send → stream → render
- Markdown + KaTeX + code highlighting

### Characters & skills (v0.3)
- SillyTavern V2/V3 PNG character card import (zero-dep PNG chunk reader)
- Skill CRUD (Markdown bodies with optional frontmatter)
- Attach card + multiple skills to an agent
- Greeting playback on first opening of a conversation

### Groups (v0.4)
- Casual group conversations (≥2 agents, capped turns)
- Work conversations with `task_goal`, per-agent turn caps, `<done/>` closes the task
- `@`-mention queueing; `<silent/>` opt-out; `<waiting/>` hands back to the user
- Wire-history tagging so each agent sees other agents' lines as user-side context

### History power tools (v0.5)
- Conversation folders
- Global message search
- Inline edit for any user/assistant message
- Regenerate creates a **variant branch** in the same `variant_group_id`
- Per-bubble swipe controls (◀ N/M ▶) to switch active variant
- All wire-history builders honor the active-variant selection

### Memory (v0.6)
- `memories` table with kind ∈ {summary, fact, preference} and importance
- One-click "💾 沉淀为记忆" calls the model with a strict JSON schema and persists each item
- Naive lexical retrieval (top-K) injected into the system prompt
- Per-agent CRUD UI 🧠

### Polish (v0.7)
- Light / dark theme toggle (CSS vars, `data-theme` attribute, instant swap)
- Markdown / JSON export
- Prompt debug Modal — see exactly what the next agent turn will receive
- zh / en i18n (library-free, Chinese keys, English overrides)

## Architectural commitments kept

- **Local-first**: every byte (DB, character cards, exports) lives on the user's disk; no telemetry, no analytics, no remote sync.
- **Single binary**: no embedded Python, no native vector DB, no second daemon. Memory retrieval is lexical; embeddings are explicitly v1.1 territory.
- **Iron law of identity**: every conversation must have exactly one user persona; private convs must have exactly one agent; group convs must have ≥2.
- **No surprise destructive ops**: regenerate creates branches instead of overwriting; deletes always confirm.
- **PolyForm NC** license — free for personal use, source-available, no commercial redistribution without permission.

## Deferred (post-1.0)

- Embedding-based memory retrieval (v1.1)
- Auto-update channel (v1.1)
- MCP tool integration (v2)
- Server / multi-user mode (v2)
- Cross-user social: A's agent visiting B's group (v3)
- Skill marketplace (v4)

## Build

```bash
pnpm install
pnpm exec vite build           # web bundle
pnpm tauri build               # desktop installers (Win / macOS / Linux)
```

The web bundle is ~900 KB minified (~275 KB gzip) — bigger than ideal because we
ship react-markdown + KaTeX + highlight.js in the main chunk. Code-splitting is
on the v1.1 cleanup list but does not affect functionality.
