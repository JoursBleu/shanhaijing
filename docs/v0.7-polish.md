# v0.7 — Polish

## What landed

### Theme switch (light / dark)
- CSS vars in `src/index.css`; light theme via `html[data-theme="light"]`
- `useUI().theme` (persisted in `localStorage["shanhaijing.theme"]`)
- Applied to `<html data-theme>` on store init and every change
- Sun/moon toggle in `AgentRail`

### Export Markdown / JSON
- `src/features/exportConversation.ts`
  - `exportConversationAsMarkdown(id)` — renders `# title` + per-turn `### speaker · role` blocks; honors active variants
  - `exportConversationAsJson(id)` — emits a self-describing payload with `schema: "shanhaijing.conversation.v1"`, conversation, persona, agents, all messages (no variant filtering — JSON is the archival format)
- Pure browser-side `<a download>` blob; no Tauri command

### Prompt debug
- `src/features/promptDebug.ts` rebuilds exactly what the next agent turn would see (primary agent for groups), incl. card / skills / memories / group section
- 🔍prompt button in conversation header opens a Modal with the raw text + copy button

### i18n (zh / en)
- `src/lib/i18n.ts` — dictionary lookup with Chinese keys as fallback; `useT()` subscribes to `useUI().language`
- Language toggle (中 / EN) in AgentRail; persisted in `localStorage["shanhaijing.language"]`
- Rail labels translated; conversation surfaces remain mostly Chinese — future PRs can add keys incrementally without breaking strings (missing keys fall back to literal)

## Design notes

- Theme uses CSS variable cascading so the entire palette swaps with zero React work; one `data-theme` attribute on `<html>` flips bg/text/border instantly.
- i18n stays library-free: keys are the Chinese source strings, so adding English is purely additive — no risk of "missing key crashed the UI".
- Prompt debug is "what the next call sees", not "what the last call saw", so it always reflects current memories/skills/cards even before sending.
- Export does not include encrypted API keys or other secrets — only conversation-scoped data.

## Known limitations

- Modal-based prompt debug doesn't auto-refresh while you type. Reopen the dialog to see the updated prompt.
- Markdown export uses ATX headings per message; long single-message exports can get heading-heavy. Acceptable for v1.0.
- i18n covers chrome only — markdown content is whatever language the model produced.
